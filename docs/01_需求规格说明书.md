# Kronos 金融预测 Web 应用 - 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
**Kronos-WebApp** - 基于 Kronos 金融模型的 Web 预测平台

### 1.2 核心功能
调用 Kronos Foundation Model 对金融 K 线数据进行价格预测可视化分析

### 1.3 目标用户
- 量化分析师
- 金融数据研究人员
- 对金融预测感兴趣的个人投资者

### 1.4 技术选型理由

| 组件 | 选择 | 理由 |
|------|------|------|
| **后端框架** | FastAPI | 异步 IO 高并发、自动 OpenAPI 文档、类型安全、Pydantic 验证 |
| **前端框架** | React 18 | 组件化生态丰富、虚拟 DOM 高性能、社区活跃 |
| **可视化库** | ECharts | 金融图表性能优秀、K线图 API 完善 |
| **任务队列** | 内存队列 | 小规模应用足矣，可扩展为 Redis |
| **部署方式** | Docker | 环境一致、便于部署 |

## 3. 数据规格

### 3.1 输入数据格式 (CSV)

**必需列**：
| 列名 | 类型 | 说明 |
|------|------|------|
| timestamps | datetime | 时间戳 |
| open | float | 开盘价 |
| high | float | 最高价 |
| low | float | 最低价 |
| close | float | 收盘价 |

**可选列**：
| 列名 | 类型 | 说明 |
|------|------|------|
| volume | float | 成交量 |
| amount | float | 成交额 |

**示例数据**：
```csv
timestamps,open,high,low,close,volume,amount
2024-06-18 11:15:00,11.27,11.28,11.26,11.27,379.0,427161.0
2024-06-18 11:20:00,11.27,11.28,11.27,11.27,277.0,312192.0
```

**预测结果 (JSON)**：
```json

  ]
}
```

## 4. API 接口设计

### 4.1 接口概览

| 方法 | 路径 | 说明 | 状态码 |
|------|------|------|--------|
| GET | /api/health | 健康检查 | 200 |
| GET | /api/models | 获取可用模型列表 | 200 |
| POST | /api/models/load | 加载指定模型 | 200/400 |
| GET | /api/models/status | 获取模型状态 | 200 |
| POST | /api/data/upload | 上传数据文件 | 200/400 |
| GET | /api/data/files | 获取已上传文件列表 | 200 |
| POST | /api/predict | 执行预测 (同步) | 200/400 |
| POST | /api/predict/async | 执行预测 (异步) | 202 |
| GET | /api/predict/{id}/status | 获取预测状态 | 200 |
| GET | /api/predict/{id}/result | 获取预测结果 | 200 |
| GET | /api/predict/history | 获取预测历史 | 200 |

### 4.2 接口详细设计

#### 4.2.1 加载模型
```
POST /api/models/load
Content-Type: application/json

Request:
{
  "model_key": "kronos-mini",    // 可选: kronos-mini, kronos-small, kronos-base
  "device": "cpu"                 // 可选: cpu, cuda:0
}

Response (200):
{
  "success": true,
  "message": "模型加载成功",
  "model_info": {
    "name": "Kronos-mini",
    "params": "4.1M",
    "context_length": 2048
  }
}
```

#### 4.2.2 上传数据
```
POST /api/data/upload
Content-Type: multipart/form-data

Form Data:
- file: [CSV 文件]

Response (200):
{
  "success": true,
  "file_id": "uuid-string",
  "file_name": "XSHG_5min_600977.csv",
  "rows": 2502,
  "preview": {
    "head": [...],
    "tail": [...]
  }
}
```

#### 4.2.3 执行预测 (异步)
```
POST /api/predict/async
Content-Type: application/json

Request:
{
  "file_id": "uuid-string",
  "lookback": 400,
  "pred_len": 120,
  "temperature": 0.8,
  "top_p": 0.9,
  "sample_count": 1,
  "start_date": "2024-06-18T11:15:00"  // 可选，指定起始时间
}

Response (202):
{
  "task_id": "uuid-string",
  "status": "pending",
  "message": "预测任务已提交"
}
```

#### 4.2.4 获取预测结果
```
GET /api/predict/{task_id}/result

Response (200):
{
  "task_id": "uuid-string",
  "status": "completed",
  "result": {
    "predictions": [...],
    "chart_config": {...},  // ECharts 配置
    "statistics": {
      "avg_close": 11.35,
      "max_close": 11.52,
      "min_close": 11.18
    }
  },
  "execution_time": 15.3
}
```

## 5. 非功能需求

### 5.1 性能需求
| 指标 | 要求 |
|------|------|
| 首次预测响应时间 | < 60 秒 (CPU) |
| 页面加载时间 | < 3 秒 |
| API 响应时间 | < 500ms (不含预测) |
| 并发预测支持 | 3 个 (CPU 限制) |

### 5.2 可用性需求
- 7x24 小时运行能力
- 模型预热：启动时自动加载默认模型
- 错误恢复：预测失败自动重试 1 次

### 5.3 安全需求
- 文件上传大小限制：50MB
- 文件类型验证：仅允许 CSV
- 数据隔离：用户只能访问自己的预测历史

## 7. 附录

### 7.1 Kronos 模型参数说明

| 参数 | 范围 | 说明 | 推荐值 |
|------|------|------|--------|
| lookback | 100-2048 | 回看窗口大小 | 400 |
| pred_len | 1-512 | 预测长度 | 120 |
| temperature (T) | 0.1-1.0 | 采样温度，越低越确定 | 0.8 |
| top_p | 0.5-1.0 | Nucleus 采样概率 | 0.9 |
| sample_count | 1-10 | 采样路径数，越多越稳定但越慢 | 1-3 |

> **注意**：使用 Kronos-mini 模型时，lookback 最大可达 2048，pred_len 最大可达 512。
