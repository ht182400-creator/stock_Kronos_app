## 问题一：日志缺少时间戳

### 问题描述
日志输出格式为 `INFO: 127.0.0.1:57124 - "POST /api/predict/async HTTP/1.1" 200 OK`，**没有显示日期和时间**，无法判断请求是否仍在继续。

### 影响范围
- 无法判断服务器是否卡住
- 无法追踪请求的时间间隔
- 难以调试长时间运行的预测任务

### 解决方案
修改 `backend/main.py`，配置日志格式添加时间戳到秒：

```python
import logging

# 配置日志格式：显示日期时间到秒
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    datefmt="%Y-%m-%d %H:%M:%S",
)
```

同时配置 uvicorn 的 access log 格式：

```python
uvicorn.run(
    "main:app",
    host=settings.HOST,
    port=settings.PORT,
    reload=settings.DEBUG,
    log_config={
        "version": 1,
        "disable_existing_loggers": False,
        "formatters": {
            "default": {
                "format": "%(asctime)s [%(levelname)s] %(message)s",
                "datefmt": "%Y-%m-%d %H:%M:%S",
            },
            "access": {
                "format": "%(asctime)s [ACCESS] %(client_addr)s - \"%(method)s %(path)s\" %(status_code)s",
                "datefmt": "%Y-%m-%d %H:%M:%S",
            },
        },
        "handlers": {
            "default": {
                "formatter": "default",
                "class": "logging.StreamHandler",
                "stream": "ext://sys.stderr",
            },
            "access": {
                "formatter": "access",
                "class": "logging.StreamHandler",
                "stream": "ext://sys.stdout",
            },
        },
        "loggers": {
            "uvicorn": {"handlers": ["default"], "level": "INFO", "propagate": False},
            "uvicorn.error": {"level": "INFO"},
            "uvicorn.access": {"handlers": ["access"], "level": "INFO", "propagate": False},
        },
    },
)
```

## 问题二：上传文件后预览显示空白

### 问题描述
上传 CSV 文件后，点击"预览 ▼"按钮展开数据预览区域，**没有显示任何数据内容**。

### 问题根因
分析代码后发现问题在于：

1. `handleUpload` 函数上传文件后：
   ```typescript
   // 旧代码 - 只获取基本信息，没有 preview
   const fileList = await dataApi.getFiles();  // getFiles() 只返回基本信息
   setFiles(fileList);
   
   const fileInfo = await dataApi.getFileInfo(response.file_id);  // 有 preview
   setSelectedFile({ ...response, ...fileInfo });  // 只更新了 selectedFile
   ```

2. `getFiles()` 返回的数据结构只有：
   ```typescript
   {
     file_id: string,
     file_name: string,
     size: number,
     uploaded_at: string
   }
   // 缺少 preview 字段！
   ```

3. `getFileInfo()` 返回完整信息包含 preview，但**没有更新 files 数组**

4. 点击"预览"时，读取的是 `files` 数组中的对象，该对象没有 `preview` 字段

### 解决方案
修改 `frontend/src/App.tsx` 中 `handleUpload` 函数，直接使用 `getFileInfo()` 返回的完整信息更新文件列表：

```typescript
// 修复后的代码
const handleUpload = useCallback(async (file: File) => {
  try {
    setLoading(true);
    const response = await dataApi.uploadFile(file);
    if (response.success) {
      // 获取完整文件信息（包含 preview）
      const fileInfo = await dataApi.getFileInfo(response.file_id);
      
      // 更新文件列表：合并基本信息 + 完整信息（含 preview）
      const newFile = { ...response, ...fileInfo };
      setFiles(prev => [newFile, ...prev]);
      
      // 选中新上传的文件
      setSelectedFile(newFile);
    }
  } catch (error) {
    console.error('上传失败:', error);
    alert('上传失败');
  } finally {
    setLoading(false);
  }
}, []);
```

### 修改文件
- `frontend/src/App.tsx`

### 验证方法
1. 重启前端服务
2. 上传一个 CSV 文件
3. 点击文件列表右侧的"预览 ▼"按钮
4. 应该显示前5条和后5条数据记录

## 问题三：数据预览功能增强

### 需求背景
原始的预览功能存在以下限制：
- 预览表格太小，限制在文件信息区域内
- 只显示前5条和后5条记录
- 无法浏览中间的数据
- 大文件（如9万+行）无法有效查看

### 用户需求
1. **扩大显示区域**：最大化预览表格的显示空间
2. **分页浏览**：上一页/下一页按钮，支持自由浏览
3. **统计信息**：显示总条数、当前页、总页数
4. **快速定位**：保留前5条和后5条的快捷查看入口
5. **翻页控制**：
   - 首页时"上一页"按钮禁用
   - 末页时"下一页"按钮禁用
   - 其他情况按钮启用

### 解决方案

#### 1. 后端增强 - 分页 API

修改 `backend/routers/data.py` 的 `GET /api/data/{file_id}` 接口：

```python
@router.get("/{file_id}")
async def get_file_info(
    file_id: str,
    page: int = 1,
    page_size: int = 5
):
    # ... 验证文件存在 ...
    
    df = pd.read_csv(file_path)
    df["timestamps"] = pd.to_datetime(df["timestamps"])
    
    total_rows = len(df)
    total_pages = (total_rows + page_size - 1) // page_size  # 向上取整
    
    # 计算分页数据
    start_idx = (page - 1) * page_size
    end_idx = start_idx + page_size
    page_data = df.iloc[start_idx:end_idx]
    
    return {
        "success": True,
        "file_id": file_id,
        "file_name": os.path.basename(file_path),
        "rows": total_rows,
        "columns": list(df.columns),
        "time_range": {
            "start": df["timestamps"].min().isoformat(),
            "end": df["timestamps"].max().isoformat(),
        },
        "preview": {
            "head": df.head(5).to_dict(orient="records"),
            "tail": df.tail(5).to_dict(orient="records"),
        },
        # 分页信息
        "pagination": {
            "page": page,
            "page_size": page_size,
            "total_rows": total_rows,
            "total_pages": total_pages,
            "has_prev": page > 1,
            "has_next": page < total_pages,
            "data": page_data.to_dict(orient="records"),
        },
    }
```

#### 2. 前端类型定义

`frontend/src/types/api.ts` 新增类型：

```typescript
export interface PaginationInfo {
  page: number;
  page_size: number;
  total_rows: number;
  total_pages: number;
  has_prev: boolean;
  has_next: boolean;
  data: Record<string, any>[];
}
```

#### 3. 前端 API Client

`frontend/src/api/client.ts` 更新 `getFileInfo` 方法：

```typescript
async getFileInfo(
  fileId: string,
  page: number = 1,
  pageSize: number = 5
): Promise<DataFileInfo & { preview?: any; pagination?: any }> {
  const response = await apiClient.get(`/api/data/${fileId}`, {
    params: { page, page_size: pageSize },
  });
  return response.data;
}
```

#### 4. 前端组件重写

`frontend/src/components/FileUpload.tsx` 完全重写，实现：

- **分页状态管理**：使用 `useState` 管理当前页、加载状态、分页数据
- **数据缓存**：使用 `useRef` 缓存已加载的页，避免重复请求
- **分页表格**：显示当前页的5条数据，带行号、交替背景色
- **分页控件**：
  - 统计信息区（页码、总条数）
  - 上一页/下一页按钮（根据 has_prev/has_next 控制禁用状态）
  - 页码跳转输入框
- **快捷查看**：保留前5条和后5条的独立展示区域
