# 预测结果问题记录

## 问题描述

Kronos 模型输出的预测结果有时不满足 OHLC 价格的基本约束：
- `high < low`（最高价低于最低价）
- `open` 或 `close` 超出 `[low, high]` 范围
- 价格偏离历史正常范围过大

## 当前处理方案

在 `backend/services/kronos_service.py` 的 `_sanitize_prediction` 方法中实现了合理化处理：

```python
# 核心逻辑：
# 1. 确保 high >= low
# 2. 确保 open 和 close 在 [low, high] 范围内
# 3. 确保价格在历史均值 ± 3σ 范围内
# 4. 价格为正数
# 5. 如果 high - low 差距太小，给予最小空间
```

详见 `backend/services/kronos_service.py:320-386`

## 用户反馈

> "模型都不准，改这些结果有意义吗？"

用户认为这是治标不治本，正确的做法应该是改进模型本身。

## 问题分析

| 角度 | 评估 |
|------|------|
| 模型改进 | 高优先级 - 根本问题所在 |
| 后处理修正 | 低价值 - 掩盖模型缺陷 |
| 保留价值 | 防止下游系统崩溃、调试参考 |

## 结论

**暂不深入研究此问题**。

合理化处理作为临时方案保留，主要用途：
1. 防止下游计算出错（技术指标、图表渲染等）
2. 作为调试参考，观察模型的具体弱点

后续如果资源允许，应优先改进模型训练策略，而非在后处理上投入更多时间。
