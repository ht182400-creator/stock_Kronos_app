# 预测结果问题记录

## 一、预测结果合理化处理

### 问题描述

Kronos 模型输出的预测结果有时不满足 OHLC 价格的基本约束：
- `high < low`（最高价低于最低价）
- `open` 或 `close` 超出 `[low, high]` 范围
- 价格偏离历史正常范围过大

### 处理方案

**已移除**。改为直接返回模型原始预测结果。

原因：
1. 合理化处理治标不治本，无法改进模型本身
2. 用户反馈"模型都不准，改这些结果没有意义"

当前实现：
- 使用 `to_float()` 函数安全转换 Tensor/numpy 类型为 Python float
- 直接返回模型原始输出，不再做后处理

## 二、Kronos RoPE 缓存线程安全问题

### 问题描述

异步预测时出现以下错误：
```
RuntimeError: unsupported operand type(s) for *: 'Tensor' and 'NoneType'
RuntimeError: The size of tensor a (401) must match the size of tensor b (400) at non-singleton dimension 2
```

### 问题根因

Kronos 模型的 `RotaryPositionalEmbedding` 类使用实例级缓存存储 RoPE 的 cos/sin 值：

```python
class RotaryPositionalEmbedding(nn.Module):
    def __init__(self, dim):
        self.cos_cached = None
        self.sin_cached = None
    
    def _update_cos_sin_cache(self, x, seq_len):
        if seq_len != self.seq_len_cached:
            # 重新计算并缓存
            self.cos_cached = ...
            self.sin_cached = ...
        return self.cos_cached, self.sin_cached
```

**问题**：
1. 多线程推理时，不同线程共享同一个模型实例
2. 缓存被多个线程同时读写，导致竞态条件
3. 第一个线程计算缓存时，其他线程可能获取到 None 或不匹配的缓存

### 错误模式观察

| 缓存值 | 当前需要 | 说明 |
|--------|----------|------|
| None | 任意 | 缓存未初始化 |
| 405 | 404 | 差 1 (lookback 不一致) |
| 401 | 400 | 差 1 |
| 519 | 400 | lookback vs lookback+pred_len |

### 解决方案

**禁用 RoPE 缓存** (`D:\Work_Area\Kronos\model\module.py`):

```python
def _update_cos_sin_cache(self, x, seq_len):
    # 禁用缓存以避免线程安全问题
    t = torch.arange(seq_len, device=x.device).type_as(self.inv_freq)
    freqs = torch.einsum('i,j->ij', t, self.inv_freq)
    emb = torch.cat((freqs, freqs), dim=-1).to(x.device)
    return emb.cos()[None, None, :, :], emb.sin()[None, None, :, :]
```

### 备选方案

1. **线程级缓存** - 使用 `threading.local()` 为每个线程维护独立缓存
2. **模型级锁** - 在预测时加锁，牺牲并发性能
3. **修复 Kronos 模型** - 重构为无状态或线程安全设计

### 影响评估

- **性能**：RoPE 计算本身很轻量，影响可忽略
- **正确性**：禁用缓存是安全的，RoPE 计算是确定性的
